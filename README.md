
# stc-save-and-return

## Glossary

| Term    | Definition                                                                                                                                         |
|---------|----------------------------------------------------------------------------------------------------------------------------------------------------|
| Complete draft   | A draft that has enough information for submission.                                                                                                |
| Draft     | Data collected from forms but not yet submitted for processing. A draft will often contain partial information, but may be complete or even empty. |
| Field | A single data entry point on a form.                                                                                                               |
| Form | A set of one or more fields designed to collect specific information.                                                                              |
| Partial draft | A draft without enough information for submission.                                                                                                 |
| Return | The process of coming back to a draft and continue entering or modifying data.                                                                     |
| Save | The process of saving the data entered by a user as they go through each form in system, even if they do not complete all the forms.               |
| Save and return | The ability to save a draft and return to it later to complete and submit.                                                                       |
| View and amend | The ability to change a what has already been submitted. Not to be confused with save and return. |

## Summary
The save and return functionality provides users with the ability to enter drafts into the STC portal and have them saved for later completion and submission.

Users can edit drafts as many times as they want before submission. This includes filling previously blank fields as well as modifying previously entered fields. The user has thirty days from initial creation of a draft to finish and submit it before it is removed.

An empty draft is created as soon as the user presses the button to start the data entry process. As the user presses the button to move from one form to the next, that data is saved. If data is entered but the button is not pressed it will be lost.
